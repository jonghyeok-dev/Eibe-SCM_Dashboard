# SCM Dashboard 에러 해결 및 개선 이력 (Error Solving & Refactoring History)

이 문서는 프로젝트 진행 중 발생한 주요 오류와 UI/UX 개선 사항, 그리고 그 해결 방법을 상세히 기록한 문서입니다. 
향후 다른 AI 에이전트나 개발자가 이 프로젝트를 이어서 작업할 때, 과거의 문제와 해결 방식을 빠르게 파악하고 동일한 실수를 반복하지 않도록 하기 위해 작성되었습니다.

---

## 1. API 무응답 (Hanging) 오류

### 🚨 문제 상황
- **증상:** `http://localhost:8000/matching` 페이지 등 주요 API 요청 시 5번 이상 시도해도 아예 무반응 상태로 로딩만 지속되며 오류 로그조차 출력되지 않는 현상 발생.
- **원인:** 백그라운드에 이전에 실행된 (혹은 비정상 종료된) Uvicorn 서버의 좀비 프로세스가 8000번 포트를 점유하고 있거나, 코드가 변경되었음에도 구버전 프로세스가 계속 실행되고 있어 발생한 데드락/포트 충돌 문제.

### 💡 해결 방법
- **조치:** 
  1. 현재 실행 중인 모든 파이썬/Uvicorn 서버 프로세스를 강제 종료(Kill)하여 포트 점유를 해제.
  2. 서버를 `python -m uvicorn app.main:app --host 0.0.0.0 --port 8000` 명령어로 깔끔하게 재시작.
  3. `/api/matching/status`, `/api/order-plan` 등의 엔드포인트가 정상적으로 `200 OK`를 반환하는지 검증 완료.
- **AI 행동 지침:** API가 무응답 상태일 경우, 코드 내부 로직을 파헤치기 전에 **가장 먼저 서버 프로세스를 완전히 죽이고 재시작**하여 포트 충돌 여부를 확인할 것.

## 2. 전역 폰트 불일치 및 테이블 레이아웃 깨짐 현상

### 🚨 문제 상황
- **증상:** 
  1. 전체적인 글꼴이 통일되지 않고, 일부 폼 요소(input, select, button)와 테이블이 브라우저 기본 폰트(굴림, 돋움 등)로 렌더링됨.
  2. 테이블 내의 데이터 문자열이 길어질 경우, 컬럼 너비가 무너지면서 UI가 깨지거나 데이터가 화면 밖으로 밀려 누락되는 현상 발생.
- **원인:** CSS에서 `body`에만 폰트를 지정하고, 폼 요소와 테이블에 명시적으로 폰트를 강제하지 않음. 테이블 셀(`td`)에 오버플로우 방지 처리가 누락됨.

### 💡 해결 방법
- **조치 (`web/static/css/style.css`):**
  1. `input, select, textarea, button, optgroup`, `table, th, td` 에 `font-family: var(--font-sans)`를 명시적으로 추가.
  2. `.data-table td` 에 `max-width: 240px`, `overflow: hidden`, `text-overflow: ellipsis` 속성을 추가하여 긴 텍스트가 말줄임표(...)로 처리되도록 방어 로직 구현.
  3. `.data-table th` 와 `td` 의 패딩 수치를 조절하여 균일한 시각적 안정성 확보.
- **AI 행동 지침:** 새로운 컴포넌트나 테이블을 추가할 때는 반드시 `text-overflow` 처리를 고려하고, 폼 요소 폰트는 전역 변수(`var(--font-sans)`)를 따르도록 주의할 것.

## 3. 발주 계획 페이지 (`order_plan.html`) UI/UX 및 구조적 버그

### 🚨 문제 상황
- **증상:**
  1. **JavaScript HTML 렌더링 버그:** "발주 이력 트렌드" 테이블(`loadOrderTrend` 함수)을 동적으로 그릴 때 `<tr>` 태그가 누락되어 구조가 깨짐.
  2. **비효율적 레이아웃:** 불필요한 배너(발주월/도착예정월/리드타임)가 상단 공간을 크게 차지함.
  3. **사용성 저하:** 시뮬레이션 결과를 보려면 매번 수량을 입력하고 "시뮬레이션 실행" 버튼을 직접 클릭해야 했음. 좌우 단 배치가 비논리적임.
- **원인:** 서버 부하가 없는 프론트엔드 연산임에도 수동 트리거(버튼) 방식을 고수함. Javascript 템플릿 리터럴 작성 시 휴먼 에러 발생.

### 💡 해결 방법
- **조치 (`web/order_plan.html`):**
  1. **배너 삭제:** `order-info-banner` HTML 요소를 완전 삭제하고 관련 JS 초기화 코드 제거.
  2. **레이아웃 논리적 재배치:**
     - **좌측:** 원인(Input)에 해당하는 [발주 수량 입력] & [발주 이력 트렌드] 배치
     - **우측:** 결과(Output)에 해당하는 [시뮬레이션 설정] & [재고일수 히트맵] & [재고 추이 차트] 배치
  3. **자동 반응형(Auto-Reactive) 전환:** "시뮬레이션 실행" 버튼을 삭제. 발주 수량 입력 필드에 `oninput="OrderPlan.onQtyChange()"` 이벤트를 추가하고, 가중치 슬라이더의 `input` 이벤트 발생 시 즉시 `runSimulation()`이 실행되도록 수정.
  4. **버그 픽스:** `loadOrderTrend()` 내의 HTML 문자열에 `<tr>` 태그를 복구하고 `colspan="7"`로 맞춤.
- **AI 행동 지침:** 클라이언트 사이드에서만 이루어지는 가벼운 시뮬레이션이나 계산은 **별도의 버튼 없이 실시간(Reactive)으로 반영**되도록 설계하여 사용자 경험을 극대화할 것. 동적 HTML 생성 시 테이블 태그 구조(`tr`, `td`, `colspan`)의 무결성을 반드시 교차 검증할 것.

## 4. Git 버전 관리 (GitHub 동기화) 이슈

### 🚨 문제 상황
- **증상:** AI를 통해 로컬 파일 시스템의 코드가 성공적으로 수정되었으나, 사용자의 GitHub 원격 저장소에는 변경 내역이 반영되지 않음.
- **원인:** AI가 로컬 파일을 직접 수정(`write_to_file`, `replace_file_content`)했으나, 이후 `git commit` 및 `git push` 절차를 밟지 않아 발생한 문제.
- **AI 행동 지침:** 핵심 기능 구현이나 버그 수정 후, 사용자가 "GitHub에 적용되지 않는다"고 지적하면 즉시 `git status`를 확인하여 변경사항을 파악하고, 사용자의 승인을 얻어 `git add`, `git commit -m "..."`, `git push` 명령어를 통해 최신 버전을 원격 저장소에 동기화할 것.
